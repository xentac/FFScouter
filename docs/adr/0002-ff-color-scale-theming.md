# Unified, discrete-bucket FF Color Scale with a reserved (UI-hidden) Custom preset

The [[FF Color Scale]] (`get_ff_arrow_colour`) drives color for four consumers — gauge arrow, gauge bubble, profile info-line background, and faction/war column background — through a single 11-step discrete-bucket lookup (`Math.floor` index into an array of 11 hex colors), not interpolation. Color theming is added as a `ColorScheme` enum (`classic` default — unchanged from today's hardcoded palette, plus `cool_diverging`, `neon`, `colorblind_safe`, `grayscale`, `custom`), selected once via `ffconfig.color_scheme` and applied identically across all four consumers — there is no per-consumer override. The no-data indicator color stays a hardcoded black regardless of scheme, since it signals absence of data rather than a point on the gradient. `custom` is reserved in the type/storage layer now (`ffconfig.custom_colors: string[] | null`, holding 11 hex strings) so a future color editor won't need a storage migration, but it's deliberately left out of the settings-panel dropdown until that editor ships; if `custom` is ever active without valid 11-entry data, it falls back to the `classic` palette.

## Considered Options

- **Per-consumer color overrides** (separate scheme for gauge marker vs. background cells): rejected — quadruples the config surface for a cosmetic feature and risks a page looking visually inconsistent with itself.
- **Switch to interpolated gradient stops** (a handful of color anchors, computed continuously): rejected — the scale and all presets are defined as 11 fixed hex buckets today; interpolation would require rewriting the lookup and re-deriving every preset's stops, and would give `custom` a different editing shape ("pick a few anchors") than the one chosen here ("edit all 11 swatches directly").
